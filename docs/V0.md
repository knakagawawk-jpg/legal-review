V0に渡す資料（YourNote「My規範・My論点」UI改善）
対象ページ
ページ: YourNote の各科目ページ（/your-page/subjects/[subject]）
対象タブ: My規範・My論点（mainTab === "study"）
前提: 左にアプリ共通サイドバー（幅約208px）があり、ページ本体は余白広め（container px-20 max-w-6xl）。全体トーンは amber系の淡いグラデ。
現状UI（要約）
画面上部に科目ドロップダウン + タブ切替（My規範・My論点 / ノート）。
My規範・My論点は 2セクション:
規範一覧（Norms）
論点一覧（Points）
各セクションは テーブルで、左端にドラッグハンドル。行は並べ替え可能。
列（現状）: 項目 / 重要度(High/Middle/Low) / 理解度(1-5) / 内容 / メモ / タグ / 作成日
空行がデフォルトで複数表示され、入力→blur/Enterで行確定する挙動（スプレッドシート風）。
折りたたみ（セクション単位）と「追加」ボタンあり。
V0で改善したいゴール
視認性: テーブルが情報過多なので、重要情報に視線誘導し、可読性を上げる
入力体験: “空行で入力”の学習コストを下げ、追加/編集が直感的になる
検索/絞り込み: 規範/論点が増えたときに探せる（タグ・重要度・理解度）
運用: 追加→編集→並び替え→復習、の一連が速い
モバイル/狭幅でも破綻しない（テーブル横スクロール依存を減らす）
デザイン方針（V0に明示）
情報構造の変更: テーブル中心 → カードリスト + 詳細ドロワー(右/下) 方式を提案
リストは「要点（項目・重要度・理解度・タグ・更新日）」に絞る
クリックで詳細（内容/メモ/作成日/タグ編集）を開く
2ペイン:
左: 規範/論点の切替 + 検索/フィルタ + 一覧
右: 選択アイテム詳細（編集フォーム）
操作の一貫性: 追加は常に上部の「＋追加」→フォームにフォーカス
視覚:
重要度はピル（High=ピンク強、Middle=ライム、Low=シアン）を踏襲
理解度は 1–5 をバッジ/バーで表現（例: 5は濃色、1は薄色）
amber背景に対して、カードは白＋薄い枠＋影少々
必須コンポーネント（V0に作ってほしいUI部品）
Segmented tabs: 「規範」「論点」切替（同一UIで運用）
Search bar: プレースホルダ例「項目・内容・メモ・タグで検索」
Filter chips:
重要度: High/Middle/Low
理解度: 1–5
タグ: 複数選択
Item list row/card（一覧1行のデザイン）:
左: ドラッグハンドル
中: 項目タイトル（太字）、サブに内容の1行抜粋
右: 重要度ピル、理解度バッジ、タグ（最大2 + “+n”）
右端: “…” メニュー（削除/作成日変更 など）
Detail drawer/panel:
項目（テキスト）
重要度（セレクト）
理解度（セレクト）
内容（textarea）
メモ（textarea）
タグ（複数、追加も可能）
作成日（datepicker）
保存/キャンセル（または自動保存前提なら保存状態インジケータ）
Empty state:
「まだデータがありません。＋追加から作成できます」
画面レイアウト案（V0向けの具体指示）
上部ヘッダー（既存を踏襲してOK）:
左: “Your Note” + アイコン
右: 科目ドロップダウン + タブ（My規範・My論点 / ノート）
My規範・My論点表示時:
さらに内側に「規範 / 論点」切替（セグメント）
その下に検索＋フィルタ列
本体は2カラム:
左カラム: リスト（スクロール）
右カラム: 詳細（スクロール、固定ヘッダーにタイトル/削除ボタン）
画面幅が狭い場合は右カラムをドロワー化（選択時にオーバーレイで開く）
インタラクション仕様（V0に伝える挙動）
追加:
「＋追加」押下で詳細パネルが“新規作成モード”になり、項目入力にフォーカス
選択:
リスト行クリックで詳細パネルに表示
並び替え:
ドラッグハンドルで並び替え（リスト上で完結）
フィルタ:
フィルタ変更で即時反映
削除:
行のメニュー or 詳細の削除ボタン（確認ダイアログ）
作成日編集:
詳細内のDatePicker（現状のPopoverカレンダー踏襲可）
UIテキスト（日本語）
セクション名: 「My規範」「My論点」
ボタン: 「＋追加」「削除」「作成日」「タグ」「メモ」
状態表示: 「読み込み中…」「履歴がありません」（※直近アクセス側で使用中の文言とも整合）
既存の雰囲気・制約（V0に必ず伝える）
カラー: amber/オレンジの淡い背景、白カード、薄いamberボーダー
文字サイズ: 小さめ（text-xs中心）でも読める余白設計
左右の余白: px-20 前提の見え方を崩さない
左サイドバーがあるので、主要操作は中央〜右寄せでも押しやすく
成果物の期待（V0への依頼文サンプル）
「YourNoteのMy規範・My論点を、スプレッドシート風テーブルから、検索・フィルタしやすい二段構え（一覧＋詳細）UIに刷新してください。amber系の既存トーンを維持し、ドラッグ並び替え、重要度/理解度/タグ/作成日編集ができる設計にしてください。モバイルでは詳細はドロワー表示にしてください。」
